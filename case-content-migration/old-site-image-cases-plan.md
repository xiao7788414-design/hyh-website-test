# 旧官网连续图片案例迁移方案

## 范围核对

- `cases.html` 中共有 31 个指向旧官网的卡片入口。
- 用户指定的 30 个品牌与旧官网页面标题逐项一致，完整映射见 `old-site-image-cases.csv`。
- 额外入口为“斯利安”（旧链接 `/catid/35/id/216.html`），不在本次明确清单内，本批保留原状。
- 30 个页面共提取 83 张正文图片，全部来自旧页 `.man_case_art` 正文容器；未抓取旧站页眉、页脚、浮动二维码或导航素材。
- 首页案例墙重复品牌共 7 个：简一瓷砖、三精蓝瓶、读客文化、好大夫在线、益佰制药、田七牙膏、晨光文具。它们与案例列表复用同一本站详情 URL。
- “六颗星长效肥”大奖卡与农产&农资卡复用 `cases/liukexing.html`。

## 信息架构

- URL：`cases/<brand-slug>.html`。
- 素材：`assets/case-details/<brand-slug>/NN.<ext>`，按旧官网正文顺序从 `01` 连续编号。
- 页面结构：返回案例列表、类别与标题、原卡片主视觉、连续案例图片、全站导航与页脚。
- 不补写旧资料中不存在的背景、项目方法、经营数据或成效；不设置“资料来源与边界”版块。
- 原有卡片分类和顺序不变；原有 `case-yibai` 锚点继续保留，其余卡片补充稳定品牌锚点。

## 英文范围

- `en/cases.html` 和 `en/index.html` 的对应入口改为中文详情页，保证不再跳旧站且链接有效。
- 本批不新增英文详情正文；30 个页面标题、类别和图片辅助文字仍为中文，列为后续翻译范围。

## 验收与发布

1. 校验 30 个卡片、7 个首页重复入口和六颗星大奖卡的一对一跳转。
2. 校验 83 张图片的文件存在、顺序、尺寸、重复和浏览器加载结果。
3. 本地桌面与 390px 手机宽度检查页面布局、返回锚点和横向溢出。
4. 同步源目录与公开发布目录，完成安全扫描、单文件大小检查及 Git 差异检查。
5. 独立提交并推送后，等待 GitHub Pages 构建 commit 与提交一致，再验收线上页面。
